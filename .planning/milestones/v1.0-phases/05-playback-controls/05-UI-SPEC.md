## Spacing Scale

Declared values (must be multiples of 4). All tokens are pre-existing — Phase 5 reuses without changes.

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Icon gaps, tight inline spacing |
| `--space-2` | 8px | Toolbar inner padding (top/bottom), BPM button gaps |
| `--space-3` | 12px | Toolbar horizontal padding (`padding: --space-2 --space-3`) |
| `--space-4` | 16px | Control group spacing inside panel |
| `--space-6` | 24px | Section gaps |
| `--space-8` | 32px | Layout-level gaps |

Exceptions:
- Toolbar controls gap: use `--space-3` (12px) between BPM widget, Volume slider, and Start/Stop button groups — matches toolbar compact density
- BPM number input width: 48px fixed (wide enough for 3-digit integers "060"–"180")
- Start/Stop button minimum width: 72px (wider than standard `btn` full-width — toolbar button is inline, not full-width)

## Color

Pre-existing palette. Phase 5 makes no new color decisions.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#111113` (`--color-bg-primary`) | App background, canvas area |
| Secondary (30%) | `#1a1a1e` (`--color-bg-secondary`) | Toolbar, cell panel sidebar |
| Tertiary surface | `#222226` (`--color-bg-tertiary`) | Control inputs, inactive beat-fraction buttons |
| Accent (10%) | `#6366f1` (`--color-accent`) | Active beat-fraction button, Start button (playing state indicator), focus rings |
| Destructive | `#ef4444` (`--color-danger`) | Remove Shape button (unchanged, no new destructive actions in Phase 5) |

Accent reserved for:
- Active/selected state on beat-fraction segmented buttons (matching `type-selector__btn--active` pattern)
- Start/Stop button border when in "Stop" (playing) state — signals active playback
- Focus-visible ring on all interactive controls (`box-shadow: 0 0 0 2px var(--color-accent)`)

Accent is NOT used for: volume slider track, BPM increment/decrement buttons, toolbar background.

#### BPM Widget

**HTML structure:**
```
<div class="toolbar-control">
  <span class="toolbar-control__label">BPM</span>
  <div class="bpm-widget">
    <button class="bpm-widget__btn" aria-label="Decrease BPM">−</button>
    <input class="bpm-widget__input" type="number" min="60" max="180" step="1" aria-label="BPM" />
    <button class="bpm-widget__btn" aria-label="Increase BPM">+</button>
  </div>
</div>
```

**Styling:**
- `.toolbar-control`: `display:flex; flex-direction:column; align-items:center; gap: --space-1`
- `.toolbar-control__label`: `font-size: --text-xs; color: --color-text-secondary; font-weight: --weight-regular`
- `.bpm-widget`: `display:flex; flex-direction:row; align-items:center; gap: --space-1; height: 28px`
- `.bpm-widget__btn`: `width: 20px; height: 20px; border-radius: --radius-sm; border: 1px solid --color-border-secondary; background: --color-bg-tertiary; color: --color-text-secondary; font-size: --text-base; cursor:pointer; display:flex; align-items:center; justify-content:center`
- `.bpm-widget__btn:hover`: `border-color: --color-border-primary; background: rgba(255,255,255,0.04)`
- `.bpm-widget__input`: `width: 48px; height: 28px; text-align:center; font-size: --text-base; font-weight: --weight-medium; color: --color-text-primary; background: --color-bg-tertiary; border: 1px solid --color-border-secondary; border-radius: --radius-sm; font-variant-numeric: tabular-nums`
- `.bpm-widget__input:focus-visible`: `outline: none; box-shadow: 0 0 0 2px --color-accent`

**Interaction:**
- `−` button: decrement by 1, clamped to 60 minimum
- `+` button: increment by 1, clamped to 180 maximum
- Keyboard on number input: arrow-up increments, arrow-down decrements (native behavior)
- Typing: direct keyboard entry, validated on blur — out-of-range values snap to nearest bound

#### Volume Widget

**HTML structure:**
```
<div class="toolbar-control">
  <span class="toolbar-control__label">Volume</span>
  <div class="slider-wrap toolbar-slider">
    <div class="slider-wrap__track" />
    <input type="range" min="0" max="1" step="0.01" aria-label="Master volume" />
  </div>
</div>
```

**Styling:**
- Reuses `.slider-wrap` and `input[type="range"]` CSS from `src/styles/index.css` (established in Phase 4) without modification
- `.toolbar-slider`: `width: 80px` — compact inline slider width for toolbar density
- Track background: `linear-gradient(to right, --color-bg-tertiary, --color-text-secondary)` (neutral, no hue — pure loudness control)

**Interaction:**
- Continuous update on `onChange` (same as HsvSliders pattern)
- Maps to `playbackStore.volume` (0–1); audio engine applies `masterGain.gain.setTargetAtTime(v * 0.15, ctx.currentTime, 0.05)` internally
- Default value: 0.8 (pre-populated from `playbackStore` default)

#### Start/Stop Button

**HTML structure:**
```
<button class="btn btn--start-stop" aria-label="Start playback" aria-pressed={isPlaying}>
  {isPlaying ? 'Stop' : 'Start'}
</button>
```

**Styling:**
- `.btn--start-stop`: `width: auto; min-width: 72px; height: 28px; padding: 0 --space-3; border-radius: --radius-sm; font-size: --text-base; font-weight: --weight-medium; cursor:pointer; border: 1px solid; display:inline-flex; align-items:center; justify-content:center`
- **Stopped state** (label "Start"): `border-color: --color-accent; color: --color-accent; background: transparent`
- **Stopped state hover**: `background: rgba(99,102,241,0.08)`
- **Playing state** (label "Stop"): `border-color: --color-border-primary; color: --color-text-primary; background: rgba(255,255,255,0.06)`
- **Playing state hover**: `background: rgba(255,255,255,0.10)`
- Focus-visible: `box-shadow: 0 0 0 2px --color-accent; outline: none`

**Interaction:**
- Single click toggles between Start and Stop
- Label changes synchronously with `isPlaying` state in `playbackStore`
- `aria-pressed` reflects playing state for screen readers

### Modified: Beat-Fraction Selector (replaces animRate Hz slider in `CellPanel`)

**Replaces:** The `<div class="control-group">` block containing `slider-anim-rate` in `CellPanel.tsx`.

**HTML structure:**
```
<div class="control-group">
  <div class="control-group__label-row">
    <label class="control-group__label">Rate</label>
    <span class="control-group__readout">{computed Hz, e.g. "2.0 Hz"}</span>
  </div>
  <div class="beat-selector" role="group" aria-label="Animation rate">
    <button class="beat-selector__btn [beat-selector__btn--active]" aria-pressed={…}>1/1</button>
    <button class="beat-selector__btn [beat-selector__btn--active]" aria-pressed={…}>1/2</button>
    <button class="beat-selector__btn [beat-selector__btn--active]" aria-pressed={…}>1/4</button>
    <button class="beat-selector__btn [beat-selector__btn--active]" aria-pressed={…}>1/8</button>
    <button class="beat-selector__btn [beat-selector__btn--active]" aria-pressed={…}>1/16</button>
  </div>
</div>
```

**Styling:**
- `.beat-selector`: `display:flex; flex-direction:row; gap: --space-1; width: 100%`
- `.beat-selector__btn`: `flex:1; height: 28px; border-radius: --radius-sm; border: 1px solid --color-border-secondary; background: --color-bg-tertiary; color: --color-text-secondary; font-size: --text-xs; font-weight: --weight-regular; cursor:pointer`
- `.beat-selector__btn:hover`: `border-color: --color-border-primary; background: rgba(255,255,255,0.04)`
- `.beat-selector__btn--active`: `border-color: --color-accent; background: rgba(99,102,241,0.12); color: --color-text-primary`
  - Matches `.type-selector__btn--active` visual pattern exactly

**Readout:** Shows computed LFO frequency: `(bpm / 60) * fractionValue` formatted to 1 decimal place + " Hz". Example at 120 BPM with 1/4 selected: "2.0 Hz". Source: `playbackStore.bpm` + selected fraction.

**Interaction:**
- Single click selects a fraction — mutually exclusive (only one active at a time)
- `aria-pressed` on each button reflects selection state
- No drag, no range — discrete selection only

## Known Pitfalls

1. **BPM number input appearance reset** — `<input type="number">` renders native spin arrows on most browsers. Hide with CSS: `-webkit-appearance:none; -moz-appearance:textfield` to keep toolbar compact. Confirm arrows are not needed since +/− buttons replace them.

2. **Start button auto-start default** — `playbackStore.isPlaying` defaults to `true` (per D-14). On first paint the button label is "Stop", not "Start". This is intentional (preserves current auto-play behavior). Do not treat this as a bug during implementation.

3. **Volume slider track gradient** — Unlike HSV sliders, the volume slider track does not need a dynamic gradient. Use a static `linear-gradient(to right, --color-bg-tertiary, --color-text-secondary)` — or simply the existing `--color-bg-tertiary` solid track. The thumb position communicates value.

4. **Beat-fraction readout depends on BPM** — The "2.0 Hz" readout in the panel must read `playbackStore.bpm` via a React hook (`usePlaybackStore`). If `playbackStore` is not yet in React context when `CellPanel` renders, the readout should fall back to the default 120 BPM value silently.
