## Playhead Visual Specification

This section documents the canvas drawing contract for the playhead line.

### Geometry

| Property | Value | Source |
|----------|-------|--------|
| Line width | 2px | CONTEXT.md D-06, REQUIREMENTS.md ANIM-14 |
| Line style | solid (no dash) | CONTEXT.md D-06 |
| Orientation | vertical — full canvas height, x = playhead pixel position | REQUIREMENTS.md ANIM-14 |
| x = 0 when stopped | left edge of lane canvas | CONTEXT.md D-05 |
| x during playback | `(beatPos % curve.duration) / curve.duration * canvasWidth` | REQUIREMENTS.md ANIM-14 |

### Draw order (within `drawLaneCanvas`)

1. Clear canvas (`clearRect`)
2. Fill background (#111113)
3. Draw midpoint dashed baseline (rgba(255,255,255,0.10), 1px, [4,4])
4. Draw curve polyline (#6366f1, 1.5px)
5. Draw control points (selected: #6366f1 r=6, unselected: rgba(255,255,255,0.55) r=5)
6. **Draw playhead line** (#6366f1, 2px, full height) — drawn last, always on top

Source: CONTEXT.md Claude's Discretion — "drawn last (on top of the curve polyline and control points)"

### Function signature extension

`drawLaneCanvas` gains one optional parameter:

```
drawLaneCanvas(ctx, w, h, curve, property, selectedPointIdx, playheadBeat?)
```

- `playheadBeat?: number` — when provided, draws the playhead at
  `x = (playheadBeat % curve.duration) / curve.duration * w`
- When undefined (or when stopped and beat = 0), draw at `x = 0`

Source: CONTEXT.md code_context — "Phase 8 extends it with an optional `playheadBeat?: number` parameter"

## Interaction Contract

### RAF Loop Lifecycle

| State | Behavior |
|-------|----------|
| `isPlaying = true` | RAF loop starts, redraws all lane canvases every frame with current beat position |
| `isPlaying = false` | RAF cancels, all lane canvases redrawn once at beat 0 (left edge) |
| Component unmounts | `cancelAnimationFrame(rafId)` called via useEffect cleanup |

Source: CONTEXT.md D-04 — "The RAF loop only runs when `isPlaying` is true"
Source: CONTEXT.md D-03 — "each frame, the loop redraws the entire lane canvas"

### React State Interaction Contract

The playhead position MUST NOT trigger React re-renders. The following rules apply:

- Beat position read via `playbackStore.getState()` (vanilla Zustand) — no hook inside RAF
- Canvas draw operations are imperative Canvas2D API calls only
- No `useState` or `setState` calls within the RAF frame callback
- `isPlaying` transition handled once via `useEffect` dependency on the React state value

Source: REQUIREMENTS.md ANIM-15 — "no React state update on each frame"
Source: STATE.md — "[v1.1]: Beat clock will be RAF-derived from AudioContext.currentTime — no playbackStore update to avoid render thrashing"

### Polyrhythm contract

Each lane shows its own phase independently. The playhead x position for lane N is:

```
beatPos % lane.curve.duration
```

Lanes with different durations will show their playhead at different x positions simultaneously.
This is the intended polyrhythm behavior — not a bug.

Source: CONTEXT.md Specific Ideas — "The playhead modulo is per-lane: `beatPos % curve.duration`"

## States

### Lane canvas states

| State | Visual |
|-------|--------|
| No curves defined | Empty state text shown — no canvas drawn — unchanged from Phase 7 |
| Stopped, no curves | Not applicable |
| Stopped, curves defined | Canvas drawn with curve + control points + playhead at x=0 |
| Playing, curves defined | Canvas redraws every RAF frame; playhead sweeps left-to-right, wraps at curve boundary |
| Playing, panel collapsed (≤40px) | RAF still runs; canvas redraws continue (panel hidden by overflow:hidden but RAF loop continues) |
