#### Hue Scale Grid Lines (ANIM-13)

| Element | Color | Opacity | Weight | Notes |
|---------|-------|---------|--------|-------|
| Root note line | `hsl(${hue}, 100%, 60%)` | 60% (`globalAlpha`) | 1.5px | Brighter, thicker тАФ D-09 |
| Non-root note lines | `hsl(${hue}, 100%, 60%)` | 28% (`globalAlpha`) | 1px | Dimmer тАФ D-09 |
| Note name labels (focused only) | `hsl(${hue}, 100%, 70%)` | 85% | n/a | Slightly lighter for legibility at 10px |

Opacity rationale (Claude's discretion тАФ D-09): Root at 60% is clearly legible against the dark canvas background (`#111113`) without competing with the curve polyline. Non-root at 28% is visible for context but recedes clearly behind the root and curve. The 2x ratio between root and non-root maintains the hierarchy at any zoom level.

The note line color is the **actual hue of the note** тАФ `hsl(${hue}, 100%, 60%)` тАФ where `hue = (semitone / 12) * 360`. This is the inverse of the `audioEngine.ts` formula and is computed by `src/engine/noteHue.ts` (new file, D-06).

Only lines within the current Y viewport range are drawn. Guard: `if (noteHue >= yMin && noteHue <= yMax)` before each draw call. Source: REQUIREMENTS.md ANIM-13.

Source: CONTEXT.md D-09; RESEARCH.md Pattern 3 (hue-colored horizontal line code example).

## Interaction Contract

### Y-Axis Scroll / Zoom (ANIM-10)

| Gesture | Modifier | Effect | Clamp |
|---------|----------|--------|-------|
| Scroll wheel | none | Pan Y viewport up/down | `yMin тЙе 0`, `yMax тЙд fullMax`; window size preserved |
| Scroll wheel | Ctrl or Cmd | Zoom Y viewport (shrink/grow visible range) | Min visible range: 10 units (non-hue), 30 units (hue); Max = full range |

Pan behavior:
- Pan amount = `Math.sign(deltaY) * Math.min(Math.abs(deltaY), 50) * 0.10 * range`
  - Delta cap of 50 normalizes trackpad (many small floats) vs mouse wheel (large integers ~100). Source: RESEARCH.md "Don't Hand-Roll" table + Assumption A3.
  - 10% of visible range per normalized unit gives a medium-speed pan on both input types.
- `newMin = Math.max(fullMin, Math.min(fullMax - range, current.min + panAmount))`
- `newMax = newMin + range` (preserves window size exactly)

Zoom behavior (Claude's discretion тАФ D-16):
- Zoom amount: 10% of current range per wheel click (`ZOOM_FACTOR = 0.10`)
- Zoom pivot: midpoint of current visible range (not cursor position тАФ simpler, deferred per CONTEXT.md deferred)
- `newRange = clamp(range ┬▒ delta, MIN_RANGE, fullRange)`
- `newMin = max(fullMin, mid - newRange/2)`; `newMax = min(fullMax, mid + newRange/2)`

Event wiring: imperative `addEventListener('wheel', handler, { passive: false })` inside `useEffect` on the lane canvas element. React's synthetic `onWheel` is passive and cannot call `preventDefault()`. Source: RESEARCH.md Pitfall 1, Pattern 4.

State storage: `uiStore.yViewport: Partial<Record<AnimatableProperty, { min: number; max: number }>>`. Absent key = full range default. Source: CONTEXT.md D-11, D-12.

No Y-axis scrollbar or thumb indicator. Deferred per CONTEXT.md deferred section and REQUIREMENTS.md future section.

### Y-Axis Viewport Indicator (Claude's discretion)

A thin left-edge indicator strip is **included** when the Y viewport is zoomed (i.e., when `yViewport[prop]` is present and range < fullRange). Rationale: without any indicator, the user has no visual feedback that the Y-axis is zoomed in, which makes the interaction model confusing. The indicator is minimal:

- **Position:** 3px-wide vertical strip on the left edge of the lane canvas column (`anim-lane__canvas-col`)
- **Color:** `rgba(99, 102, 241, 0.40)` тАФ accent at 40% opacity (matches accent color `--color-accent`)
- **Thumb position:** proportional to `yMin / fullMax` from bottom (canvas Y is inverted тАФ high values at top)
- **Thumb height:** proportional to `visibleRange / fullRange * canvasHeight`
- **Implementation:** drawn in `drawLaneCanvas` as a canvas 2D rectangle in the final layer (after playhead), only when `yMin > fullMin || yMax < fullMax`
- **No interaction** тАФ it is a passive indicator only. The scrollbar deferred in REQUIREMENTS.md refers to an interactive scrollbar; this is a read-only indicator.
