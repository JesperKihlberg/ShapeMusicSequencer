| Interaction | Behavior |
|-------------|----------|
| cursor | pointer (new on .anim-lane__label-col) |
| click when no lane focused | focus this lane; all others compress |
| click on already-focused lane | collapse — no lane focused; all return to default height |

### ResizeObserver behavior on height change

The existing `ResizeObserver` in `AnimLane` already watches the canvas container div.
When `.anim-lane--focused` or `.anim-lane--compressed` snaps the lane to a new height,
the ResizeObserver fires, updates `canvas.width` and `canvas.height`, and the next draw
call fills the new dimensions. No additional wiring required.

Source: CONTEXT.md `<code_context>` — "ResizeObserver in AnimLane already handles canvas
resize on height change — the class-driven height change will trigger it automatically"

## Ghost Repetitions Specification (ANIM-09)

### Rendering contract

Ghost copies are drawn in the RAF loop (and stopped-state draw) by the caller of
`drawLaneCanvas`, not inside `drawLaneCanvas` itself.

Draw order per lane per frame:

1. Draw primary region: `drawLaneCanvas(ctx, primaryWidthPx, h, curve, property, selectedIdx, playheadBeat)`
2. For each ghost copy `i = 1` to `repeatCount` (where `repeatCount = Math.floor(zoomBeats / curve.duration) - 1`):
   - `ctx.save()`
   - `ctx.globalAlpha = 0.30`
   - `ctx.beginPath(); ctx.rect(ghostStartPx, 0, ghostWidthPx, h); ctx.clip()`
   - `ctx.translate(ghostStartPx, 0)`
   - `drawLaneCanvas(ctx, ghostWidthPx, h, curve, property, null, undefined)` — no selectedIdx, no playhead
   - `ctx.restore()`
3. Optional partial ghost at right edge: same clip+translate approach with a narrower clip rect
   when `zoomBeats % curve.duration !== 0`. This is the natural fallout of the clip approach.

Ghost pixel geometry:

| Value | Formula |
|-------|---------|
| primaryWidthPx | `(curve.duration / zoomBeats) * canvasWidth` |
| ghostStartPx (copy i) | `(i * curve.duration / zoomBeats) * canvasWidth` |
| ghostWidthPx | Same as primaryWidthPx (for full copies) |

### Interaction exclusion contract

Canvas pointer events (pointerdown, pointermove) in `AnimLane` check:
- If `pixelX > primaryRegionWidth` (i.e. `pixelX > (curve.duration / zoomBeats) * canvas.width`),
  the click is in a ghost region. Ignore the event (no point insertion, no hit-test).
- If `pixelX <= primaryRegionWidth`, proceed with existing hit-test logic.

Source: CONTEXT.md D-08, D-09, D-10
Source: REQUIREMENTS.md ANIM-09 — "ghost regions are non-interactive (no click, drag, or point insertion)"

### Optional separator line

A 1px vertical line at `x = primaryRegionWidth` between the primary region and the first
ghost may be drawn if it aids readability. Color: rgba(255,255,255,0.15). Drawn after the
primary region, before the first ghost pass. Claude decides at implementation time whether
to include it.

Source: CONTEXT.md Claude's Discretion, REQUIREMENTS.md "optional for v1.1"

## drawLaneCanvas Signature Extension

The function gains an optional `zoomBeats` parameter (Phase 9 addition):

```
drawLaneCanvas(ctx, w, h, curve, property, selectedIdx, playheadBeat?, zoomBeats?)
```

- `zoomBeats?: number` — when provided, the X-axis spans `zoomBeats` beats instead of
  `curve.duration` beats. The `toPixel` mapping becomes:
  `px = (p.beat / zoomBeats) * w` (not `/ curve.duration`)
- When `zoomBeats` is not provided (or `zoomBeats === curve.duration`), behavior is unchanged.
- The primary region caller passes `zoomBeats = curve.duration` equivalent (unchanged behavior).
  Ghost pass callers pass `zoomBeats = undefined` (function uses `curve.duration` internally
  to fill the ghost canvas width proportionally — ghost width IS proportional to curve duration).

Source: CONTEXT.md D-07, D-09

## Interaction Contract

### Zoom control

| Interaction | Behavior |
|-------------|----------|
| Click a zoom button | `uiStore.setZoomBeats(value)`; button gets `--active` class; all lane canvases redraw next RAF frame |
| Active button on load | "4" (zoomBeats default = 4) |
| No animation | Instant — no transition |
| Keyboard | Each button is `<button>` — Space/Enter activate; focus ring via box-shadow |

### Lane focus toggle

| Interaction | Behavior |
|-------------|----------|
| Click label column (unfocused lane, no focused lane) | Set focusedLane = this property; lane snaps to 160px; others compress to 44px |
| Click label column (unfocused lane, different lane focused) | Set focusedLane = this property; this lane snaps to 160px; previous focused lane compresses to 44px |
| Click label column (already focused lane) | Set focusedLane = null; all lanes return to default height (min-height: 56px) |
| First load | No lane focused — all lanes at default height |
| No transition | Height change is instant; no CSS transition |
| Keyboard | Label column is not a `<button>` by default — consider `role="button"` + `tabIndex={0}` + `onKeyDown` handler for Enter/Space |

### Ghost region pointer exclusion

| Event | Ghost region behavior |
|-------|----------------------|
| pointerdown | Ignored — no point insertion, no drag start |
| pointermove | Ignored — no drag |
| click | Falls through to canvas element; exclusion is in JS hit-test logic |

### Zoom button states

| State | Visual |
|-------|--------|
| Resting | bg #222226, border rgba(255,255,255,0.10), text rgba(255,255,255,0.55) |
| Hover | bg rgba(255,255,255,0.04), border rgba(255,255,255,0.18) |

### Lane canvas states (additions to Phase 8 contract)

| State | Visual |
|-------|--------|
| zoomBeats > curve.duration | Primary region drawn at reduced width; ghost copies fill remainder at 0.30 opacity |
| zoomBeats = curve.duration | No ghosts drawn; primary fills full canvas width (same as Phase 8) |
| zoomBeats < curve.duration | Canvas shows only the first zoomBeats of the curve; no ghosts |
| Lane focused (160px height) | Canvas taller; ResizeObserver updates canvas.height; curve redraws at full opacity |
| Lane compressed (44px height) | Canvas shorter; curve visible but compressed; all at full opacity (no special treatment) |
