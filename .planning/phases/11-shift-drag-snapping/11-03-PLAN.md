<objective>
Fix the snapped visual rendering gap from VERIFICATION.md Truth 6.

Purpose: The snapped control-point visual (white fill + #6366f1 ring) does not render during Shift+drag
because `isSnappedRef.current` is a React ref — changing `.current` does not trigger the static-draw
useEffect to re-fire. The effect captures the ref value at scheduling time, so the `curve` change from
`setCurve` fires the effect, but by the time effects flush the ref may have already been read stale in
the closure snapshot. Converting snap state to a dual pattern — keep `isSnappedRef` for the RAF loop
(already works via `laneSnappedRefs`) AND add a mirrored React state variable `isSnapped` that is added
to the static-draw useEffect dependency array — guarantees the effect re-fires whenever snap state
changes and reads the current value.

Output: Modified `src/components/AnimationPanel.tsx` with `isSnapped` React state declared alongside
`isSnappedRef`, all 5 setter sites updated to call both `isSnappedRef.current = X` and `setIsSnapped(X)`,
`primaryOptions.isSnapped` reading from state (not ref), and `isSnapped` in the static-draw dependency array.
</objective>
