# Quick Task 260428: when first adding an animation curve the curve is not drawn

## Root Cause

`AnimLane` has two separate effects:
1. **ResizeObserver** (line 468): sets `canvas.width`/`canvas.height` from container
2. **Draw effect** (line 480): draws curve, depends on `[curve, property, selectedPointIdx]`

When a curve is first added:
1. `AnimLane` mounts → canvas is 0×0 (default)
2. Draw effect fires (curve changed) → draws to 0×0 canvas → nothing visible
3. ResizeObserver fires → sets real dimensions → but draw effect does NOT re-run (canvas size not in deps)

Result: curve drawn to 0×0 then never redrawn until next `curve`/`property`/`selectedPointIdx` change.

## Fix

In `AnimLane`'s ResizeObserver callback, after resizing the canvas, call a redraw. The cleanest approach is to extract the draw logic into a `drawRef` callback that both effects can call, or simply re-invoke the draw inline after resize using refs.

Since the draw effect already captures `curve`, `property`, `selectedPointIdx` via closure, the simplest fix is to move the draw into a `useCallback` stored in a ref so the ResizeObserver can call it.

Alternatively (simpler): add a `canvasSize` state that the ResizeObserver sets, and include it in the draw effect's deps. This forces a redraw whenever dimensions change.

## Tasks

### Task 1: Fix AnimLane to redraw after canvas resize

**File:** `src/components/AnimationPanel.tsx`
**Action:** Add `canvasSize` state (width/height) to `AnimLane`. ResizeObserver sets it; draw effect depends on it — ensuring redraw fires after first mount resize.

**Implementation:**
- Add `const [canvasSize, setCanvasSize] = useState({ w: 0, h: 0 })` inside `AnimLane`
- In ResizeObserver callback, after setting canvas dimensions, call `setCanvasSize({ w: container.clientWidth, h: container.clientHeight })`
- Add `canvasSize` to draw effect deps: `[curve, property, selectedPointIdx, canvasSize]`

**verify:** First-add curve renders immediately; subsequent curve changes still render.
**done:** `canvas.width > 0` when draw fires on first add.
